서브쿼리
	단일행 서브쿼리 ; 서브쿼리의 결과가 단일 행
			단일 행 연산자 : =, >, >=, <, <=, <>
	다중행 서브쿼리 ; 서브쿼리의 결과가 2행 이상의 행
			복수 행 연산자 : IN, NOT IN, ANY, SOME, ALL, EXISTS


서브쿼리란 ? 
	하나의 SQL 문장의 절 안에 포함된 또 하나의 SELECT 문장
	서브쿼리를 포함하고 있는 쿼리문을 메인쿼리, 포함된 쿼리를 서브쿼리라고 부른다.
	서브 쿼리는 비교 연산자의 오른쪽에 기술하고 반드시 괄호로 둘러야한다.
	서브 쿼리는 메인 쿼리가 실행되기 이전에 한번만 실행이 된다.


1)단일행 서브쿼리
	수행 결과 오직 하나의 행(row)만을 반환하는 서브 쿼리를 갖는 것
	하나의 행으로 반환된 서브 쿼리의 결과가 메인 쿼리로 보내지고, 메인 쿼리의 WHERE절에서 단일 행 비교 연산자인 =, >, >=, <, <=, <>를 사용하게 된다.


2)다중행 서브쿼리
	수행 결과 하나 이상의 행을 반환하는 서브 쿼리를 갖는 것.
	다중 행 서브 쿼리는  다중 행 연산자(Multiple Row Operator)와 함께 사용하게된다.
	다중 행 연산자의 종류:
		IN : 메인 쿼리의 비교 조건
		   :(‘=’ 연산자로 비교할 경우)이 서브 쿼리의 결과 중에서 하나라도 일치하면 참
		ANY, SOME : 메인 쿼리의 비교 조건이 서브 쿼리의 검색 결과와 하나 이상이 일치하면 참
		ALL : 메인 쿼리의 비교 조건이 서브 쿼리의 검색 결과와 모든 값이 일치하면 참
		EXISTS : 메인 쿼리의 비교 조건이 서브 쿼리의 결과 중에서 만족하는 값이 하나라도 존재하면 참
